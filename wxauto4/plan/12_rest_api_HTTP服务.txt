通用背景：
你正在改造 SuperWx / wxauto4，使其成为一个基于 Windows 微信 4.x UIAutomation 的本地微信 AI 自动化框架。
当前实测核心控件如下：

main_window:
- ClassName: mmui::MainWindow
- Name: 微信 / Weixin / WeChat

chat_message_page:
- AutomationId: chat_message_page
- ClassName: mmui::ChatMessagePage

message_view:
- ClassName: mmui::MessageView

chat_message_list:
- AutomationId: chat_message_list
- ClassName: mmui::RecyclerListView
- Name: 消息

input_view:
- ClassName: mmui::InputView

chat_input_field:
- AutomationId: chat_input_field
- ClassName: mmui::ChatInputField
- ControlType: EditControl
- 禁止用 Name 定位，因为 Name 会变化

send_button:
- Name: 发送
- ClassName: mmui::XOutlineButton
- ControlType: ButtonControl
- 禁止把“发送表情”“发送文件”“发送收藏”误判为发送消息按钮

session_list:
- AutomationId: session_list
- ClassName: mmui::XTableView
- Name: 会话

全局限制：
1. 不要修改授权、license、auth、加密、激活、网络校验相关代码。
2. 不要删除旧逻辑，只能新增 fallback 或兼容逻辑。
3. 不要一次性跨模块乱改。
4. 每个模块完成后必须输出：修改了哪些文件、改了什么、测试命令、测试结果、是否可以进入下一个模块。
5. 高风险操作默认 dry_run，不得默认真实发送、加好友、改群、发朋友圈。
6. 如果无法确认某功能，不要编造，输出检测脚本和失败原因。
7. 所有测试脚本默认不真实发送、不真实修改微信状态，除非用户显式传 --send、--apply、--confirm。

# 模块 12：RESTful HTTP API 服务

目标：
用 FastAPI 把 wxauto4 能力暴露成本地 HTTP API，方便 Hermes-Agent、OpenClaw、网页面板调用。

新增文件：
1. wxauto4/api/__init__.py
2. wxauto4/api/main.py
3. wxauto4/api/schemas.py
4. wxauto4/api/routes/
5. tests/test_rest_api_module12.py

核心接口：
GET  /health
POST /wechat/init
GET  /wechat/status
GET  /wechat/me
GET  /wechat/sessions
POST /wechat/chat/switch
POST /wechat/message/send
POST /wechat/file/send
GET  /wechat/messages/current
GET  /wechat/messages/new
POST /wechat/listen/add
POST /wechat/listen/remove
POST /wechat/listen/start
POST /wechat/listen/stop
POST /wechat/doctor

返回格式：
{
  "ok": true,
  "data": {},
  "error": null
}

安全要求：
1. 默认只监听 127.0.0.1。
2. 可选 API token。
3. 发送类接口支持 dry_run。
4. 文件路径必须校验存在。
5. 不允许远程任意文件读取。

启动命令：
python -m wxauto4.api.main

测试：
python tests/test_rest_api_module12.py
