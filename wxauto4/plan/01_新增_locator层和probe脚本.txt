项目背景：
你正在改造 SuperWx / wxauto4，使其适配当前 Windows 微信 4.x UI。
当前已通过 UIAutomation 实测确认以下核心 selector：

main_window:
- ClassName: mmui::MainWindow
- Name 可能为：微信 / Weixin / WeChat

chat_message_page:
- AutomationId: chat_message_page
- ClassName: mmui::ChatMessagePage

message_view:
- ClassName: mmui::MessageView

chat_message_list:
- AutomationId: chat_message_list
- ClassName: mmui::RecyclerListView
- Name: 消息

input_view:
- ClassName: mmui::InputView

chat_input_field:
- AutomationId: chat_input_field
- ClassName: mmui::ChatInputField
- ControlType: EditControl
- 注意：不要用 Name 定位，因为 Name 会变化

send_button:
- Name: 发送
- ClassName: mmui::XOutlineButton
- ControlType: ButtonControl
- 注意：不要把“发送表情”“发送文件”“发送收藏”当作发送消息按钮

session_list:
- AutomationId: session_list
- ClassName: mmui::XTableView
- Name: 会话

通用限制：
1. 不要修改授权、license、auth、加密、网络校验相关代码。
2. 不要删除旧逻辑，只能新增 fallback。
3. 不要一次性跨阶段乱改。
4. 每阶段完成后必须输出：修改了哪些文件、改了什么、测试命令、测试结果、下一阶段是否可继续。
5. 如果某功能无法确认，不要编造，先输出检测脚本和失败原因。


你现在执行第一阶段：新增 locator 层和 probe 验证脚本。

本阶段目标：
只新增通用定位层，不改现有业务逻辑，不接入 ChatBox，不改发送消息。

1. wxauto4/locator/__init__.py

engine.py 必须实现：
1. safe_get(ctrl, attr, default="")
2. rect_to_dict(ctrl)
3. ctrl_exists(ctrl)
4. try_find(root, method_name, search_depth=30, **kwargs)
5. find_first(root, selectors, search_depth=30)
6. find_wechat_window()
7. describe_control(ctrl)

probe_wechat_locator.py 必须实现：
1. 查找微信主窗口。
2. 定位 chat_message_page。
3. 定位 message_view。
4. 定位 chat_message_list。
5. 定位 input_view。
6. 定位 chat_input_field。
7. 定位 send_button。
8. 定位 session_list。
9. 打印每个控件是否 OK。
10. 不发送消息、不点击发送按钮、不修改微信状态。
11. 生成 TXT 报告，文件名格式：脚本名称_日期_时间.txt。

输出格式示例：
main_window          OK
chat_message_page    OK
message_view         OK
chat_message_list    OK
input_view           OK
chat_input_field     OK
send_button          OK
session_list         OK
