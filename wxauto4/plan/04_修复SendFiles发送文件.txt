项目背景：
你正在改造 SuperWx / wxauto4，使其适配当前 Windows 微信 4.x UI。
当前已通过 UIAutomation 实测确认以下核心 selector：

main_window:
- ClassName: mmui::MainWindow
- Name 可能为：微信 / Weixin / WeChat

chat_message_page:
- AutomationId: chat_message_page
- ClassName: mmui::ChatMessagePage

message_view:
- ClassName: mmui::MessageView

chat_message_list:
- AutomationId: chat_message_list
- ClassName: mmui::RecyclerListView
- Name: 消息

input_view:
- ClassName: mmui::InputView

chat_input_field:
- AutomationId: chat_input_field
- ClassName: mmui::ChatInputField
- ControlType: EditControl
- 注意：不要用 Name 定位，因为 Name 会变化

send_button:
- Name: 发送
- ClassName: mmui::XOutlineButton
- ControlType: ButtonControl
- 注意：不要把“发送表情”“发送文件”“发送收藏”当作发送消息按钮

session_list:
- AutomationId: session_list
- ClassName: mmui::XTableView
- Name: 会话

通用限制：
1. 不要修改授权、license、auth、加密、网络校验相关代码。
2. 不要删除旧逻辑，只能新增 fallback。
3. 不要一次性跨阶段乱改。
4. 每阶段完成后必须输出：修改了哪些文件、改了什么、测试命令、测试结果、下一阶段是否可继续。
5. 如果某功能无法确认，不要编造，先输出检测脚本和失败原因。


你现在执行第四阶段：修复 SendFiles 文件发送逻辑。

前置条件：
第三阶段 SendMsg 已经可以稳定发送文本。

本阶段目标：
让 SendFiles 能通过微信输入框粘贴文件并点击真正发送按钮。

修改范围：
- wxauto4/ui/chatbox.py
- 必要时新增 tests/test_sendfiles_stage4.py

本阶段允许改：
- SendFiles 相关函数
- 文件路径检查
- 粘贴文件到微信输入框
- 粘贴后重新定位 send_button

本阶段禁止改：
1. 消息监听。
2. ChatWith。
3. 授权 / license / auth 相关文件。
4. 不要把“发送文件”按钮当作最终发送消息按钮。

核心要求：
1. 检查文件路径是否存在。
2. 支持单文件和多文件。
3. 将文件路径写入剪贴板。
4. 点击 chat_input_field。
5. Ctrl+V 粘贴文件。
6. 等待文件卡片出现。
7. 重新定位 send_button。
8. 只点击 Name="发送" 的按钮。
9. 发送失败时输出清晰错误。

新增测试脚本 tests/test_sendfiles_stage4.py：
1. 默认 dry_run=True，不真实发送。
2. 自动生成一个临时 txt 文件，例如 tests/tmp_wxauto4_file_test.txt。
3. dry_run 时只验证文件存在、editbox 存在、可粘贴准备完成。
4. 只有传 --send-file 才真实发送。
5. 真实发送后输出结果。

测试命令：
python tests/test_sendfiles_stage4.py
python tests/test_sendfiles_stage4.py --send-file

完成后输出：
1. 修改了哪些文件。
2. SendFiles 当前流程。
3. dry_run 测试结果。
4. 是否真实发送测试。
5. 是否可以进入第五阶段。
