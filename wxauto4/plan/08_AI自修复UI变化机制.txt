项目背景：
你正在改造 SuperWx / wxauto4，使其适配当前 Windows 微信 4.x UI。
当前已通过 UIAutomation 实测确认以下核心 selector：

main_window:
- ClassName: mmui::MainWindow
- Name 可能为：微信 / Weixin / WeChat

chat_message_page:
- AutomationId: chat_message_page
- ClassName: mmui::ChatMessagePage

message_view:
- ClassName: mmui::MessageView

chat_message_list:
- AutomationId: chat_message_list
- ClassName: mmui::RecyclerListView
- Name: 消息

input_view:
- ClassName: mmui::InputView

chat_input_field:
- AutomationId: chat_input_field
- ClassName: mmui::ChatInputField
- ControlType: EditControl
- 注意：不要用 Name 定位，因为 Name 会变化

send_button:
- Name: 发送
- ClassName: mmui::XOutlineButton
- ControlType: ButtonControl
- 注意：不要把“发送表情”“发送文件”“发送收藏”当作发送消息按钮

session_list:
- AutomationId: session_list
- ClassName: mmui::XTableView
- Name: 会话

通用限制：
1. 不要修改授权、license、auth、加密、网络校验相关代码。
2. 不要删除旧逻辑，只能新增 fallback。
3. 不要一次性跨阶段乱改。
4. 每阶段完成后必须输出：修改了哪些文件、改了什么、测试命令、测试结果、下一阶段是否可继续。
5. 如果某功能无法确认，不要编造，先输出检测脚本和失败原因。


你现在执行第八阶段：实现 AI 自修复 UI 变化机制。

前置条件：
前七阶段已经完成，基础功能可用。

本阶段目标：
当微信 UI 更新导致控件失效时，程序能够：
1. 自动运行 doctor。
2. 自动 dump 当前 UI 树。
3. 自动保存失败控件信息。
4. 自动生成 repair_context.md。
5. 让 AI 根据旧 selector + 新 UI 树生成修复建议。
6. 修复优先写 selector，不要直接改业务代码。
7. 只有测试通过后才允许应用 patch。

新增建议文件：
wxauto4/locator/
  dump.py
  doctor.py
  repair_context.py
  patch_guard.py

新增目录：
.wxauto4_repair/
  dumps/
  reports/
  patches/

dump.py 实现：
1. dump 当前微信 UI 树。
2. 保存 JSON。
3. 保存 TXT。
4. 保存摘要。
5. 文件名格式：dump_wechat_ui_日期_时间.json / .txt

doctor.py 实现：
1. 检查 main_window。
2. 检查 chat_message_page。
3. 检查 chat_message_list。
4. 检查 chat_input_field。
5. 检查 send_button。
6. 检查 session_list。
7. 输出 OK / FAILED。
8. 如果失败，自动调用 dump.py。

repair_context.py：
当控件失败时生成 repair_context.md，内容包括：
1. 当前失败控件名称。
2. 旧 selector。
3. 当前 UI 树摘要。
4. 可能候选控件。
5. 限制条件：
   - 优先改 selectors.py
   - 不允许改授权代码
   - 不允许改网络代码
   - 不允许删除旧 fallback
   - 不允许把发送表情/发送文件当发送按钮
6. 要求 AI 输出 unified diff patch。

patch_guard.py：
允许修改：
- wxauto4/locator/
- wxauto4/ui/chatbox.py
- wxauto4/ui/main.py
- wxauto4/ui/sessionbox.py
- tests/

禁止修改：
- license
- auth
- wxauto4/utils/useful.py
- eval
- exec
- os.system
- 危险 subprocess
- requests.post 上传 UI 树或聊天内容

AI 修复流程：
python -m wxauto4 doctor
如果失败：
  1. dump UI 树
  2. 生成 repair_context.md
  3. 人工把 repair_context.md 喂给 AI
  4. AI 输出 patch
  5. patch_guard 检查
  6. git apply patch
  7. 重新运行 doctor
  8. 通过后提交 commit
  9. 失败则 rollback

不要全自动应用：
必须生成 patch、运行 patch_guard、运行 doctor、运行 regression，通过后人工确认提交。

测试命令：
python -m wxauto4 doctor
python -m wxauto4 doctor --dump-on-fail
python tests/regression_wechat_stage7.py

完成后输出：
1. 新增哪些文件。
2. doctor 失败时生成哪些文件。
3. repair_context.md 格式。
4. patch_guard 限制。
5. 如何让 AI 修复 UI 变化。
6. 如何回滚。
