通用背景：
你正在改造 SuperWx / wxauto4，使其成为一个基于 Windows 微信 4.x UIAutomation 的本地微信 AI 自动化框架。
当前实测核心控件如下：

main_window:
- ClassName: mmui::MainWindow
- Name: 微信 / Weixin / WeChat

chat_message_page:
- AutomationId: chat_message_page
- ClassName: mmui::ChatMessagePage

message_view:
- ClassName: mmui::MessageView

chat_message_list:
- AutomationId: chat_message_list
- ClassName: mmui::RecyclerListView
- Name: 消息

input_view:
- ClassName: mmui::InputView

chat_input_field:
- AutomationId: chat_input_field
- ClassName: mmui::ChatInputField
- ControlType: EditControl
- 禁止用 Name 定位，因为 Name 会变化

send_button:
- Name: 发送
- ClassName: mmui::XOutlineButton
- ControlType: ButtonControl
- 禁止把“发送表情”“发送文件”“发送收藏”误判为发送消息按钮

session_list:
- AutomationId: session_list
- ClassName: mmui::XTableView
- Name: 会话

全局限制：
1. 不要修改授权、license、auth、加密、激活、网络校验相关代码。
2. 不要删除旧逻辑，只能新增 fallback 或兼容逻辑。
3. 不要一次性跨模块乱改。
4. 每个模块完成后必须输出：修改了哪些文件、改了什么、测试命令、测试结果、是否可以进入下一个模块。
5. 高风险操作默认 dry_run，不得默认真实发送、加好友、改群、发朋友圈。
6. 如果无法确认某功能，不要编造，输出检测脚本和失败原因。
7. 所有测试脚本默认不真实发送、不真实修改微信状态，除非用户显式传 --send、--apply、--confirm。

# 模块 15：doctor 与 UI 自修复机制

目标：
当微信 UI 变化导致自动化失效时，自动检测失败点、dump UI 树、生成 AI 修复上下文。

新增文件：
1. wxauto4/doctor.py
2. wxauto4/locator/dump.py
3. wxauto4/locator/repair_context.py
4. wxauto4/locator/patch_guard.py
5. tests/test_doctor_module15.py

新增目录：
.wxauto4_repair/
├── dumps/
├── reports/
└── patches/

doctor 功能：
1. 检查 main_window。
2. 检查 chat_message_page。
3. 检查 chat_message_list。
4. 检查 chat_input_field。
5. 检查 send_button。
6. 检查 session_list。
7. 输出 OK / FAILED。
8. 失败时自动 dump UI 树。
9. 生成 repair_context.md。

dump.py：
1. 保存完整 UI 树 JSON。
2. 保存 TXT 摘要。
3. 保存失败控件候选列表。
4. 文件名包含日期时间。

repair_context.md 内容：
1. 失败控件名称。
2. 当前旧 selector。
3. 当前 UI 树摘要。
4. 候选控件。
5. 修改限制。
6. 要求 AI 只输出 unified diff patch。

patch_guard.py：
允许修改：
- wxauto4/locator/
- wxauto4/ui/chatbox.py
- wxauto4/ui/main.py
- wxauto4/ui/sessionbox.py
- tests/

禁止修改：
- license
- auth
- wxauto4/utils/useful.py
- eval
- exec
- os.system
- 危险 subprocess
- requests.post 上传聊天内容

测试：
python -m wxauto4 doctor
python -m wxauto4 doctor --dump-on-fail
python tests/test_doctor_module15.py
